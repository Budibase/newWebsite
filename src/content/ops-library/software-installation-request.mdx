<h2 id="overview">Overview</h2>

Assess software installation requests against approved software policy and route exceptions for review. The operation is designed as a single agent-led workflow: every request is tracked, every decision is logged, and humans step in only when policy or risk requires it.

#### Request

A requester submits a software installation request through Slack, Teams, or the Software Install Request Form, creating a tracked request row.

#### Agent Triage

The agent confirms the request type, retrieves context from software catalog and endpoint management, checks the relevant policy source, and asks for any missing details in chat.

#### Agent Resolution

Where policy permits, the agent completes approved catalog software for eligible devices and roles and records the outcome without waiting for manual handling.

#### Escalation

The agent escalates unapproved software, license conflicts, admin rights, or security review requirements to the responsible owner with a decision summary and supporting context.

#### Human Resolution

An approver reviews the escalation in chat or the Software Review Queue, then approves, rejects, or returns it for more information.

<h2 id="agents">Agents</h2>

The Software Installation Request agent manages intake, policy checks, tool actions, escalation, requester updates, and closure for this operation.

```text
# Agent role
You are the Software Installation Request agent. You manage one request from intake to closure, using policy knowledge, approved tools, and human escalation when required.

# Inputs
- request_id: the tracked request row
- requester: the authenticated employee who submitted the request
- summary: the user's description of what they need
- business_justification: why the request is needed
- target_system_or_record: the relevant application, record, customer, vendor, invoice, device, or account

# Instructions
1. Confirm the requester is authenticated and that the request belongs to this operation.
2. Gather missing details in Slack or Teams before taking action. Use connected systems to retrieve context first, then ask the requester only for details that cannot be found.
3. Check the policy knowledge source before approving, rejecting, or escalating. Do not invent policy rules.
4. Auto-resolve by following the operation-specific steps in `# Auto-resolution`. Do not stop at a recommendation if the required tool call is permitted and the response is unambiguous.
5. Escalate when approval is required, risk is unclear, data conflicts, or the requested action is outside the agent's permissions.
6. Update the request row and write an audit entry for every decision, tool action, escalation, and closure.
7. Reply to the requester with a concise status update and next step.

# Tool use
- Use {{ budibase.Software Installation Requests.get_row }} and {{ budibase.Software Installation Requests.update_row }} to maintain request state.
- Use {{ budibase.Software Installation Audit Log.create_row }} to log decisions and actions.
- Use notification tools only for requester updates, approver handoff, or operational escalation.

# Auto-resolution
- Use {{ endpointManagement.get_device }} to confirm the requester owns the target device.
- Use {{ endpointManagement.get_software_catalog_item }} to confirm the software is approved and available.
- If policy allows installation, use {{ endpointManagement.deploy_software }} to push the package to the device.
- Use {{ jira.create_issue }} for manual packaging, license purchase, or security-review exceptions.
- Reject requests for unapproved software when policy explicitly blocks installation.

# Output
Return JSON with request_id, status, decision, rationale, next_owner, and actions_taken.
```

<h2 id="data">Data</h2>

### Tables

**Software Installation Requests**: Stores the request, requester, target record, status, current owner, decision, and closure details.

- `request_id` : Text - Unique request identifier.
- `requester` : User - Authenticated employee who submitted the request.
- `summary` : Long Form Text - Short description of the request.
- `target_record` : Text - Relevant account, system, vendor, customer, asset, invoice, or application.
- `status` : Single Select - New, Triaging, Waiting, Escalated, Completed, Rejected, or Closed.
- `priority` : Single Select - Low, Medium, or High.
- `decision` : Single Select - Approved, Rejected, Escalated, or Cancelled.
- `rationale` : Long Form Text - Agent or approver explanation.
- `created_at` : Date - Request creation timestamp.
- `closed_at` : Date - Completion timestamp, if closed.

```csv
request_id,requester,summary,target_record,status,priority,decision,rationale,created_at,closed_at
IT-1048,emma.clarke@example.com,"Please process this software installation request.","Example target",Escalated,High,Escalated,"Requires owner approval.",2026-05-18T09:15:00.000Z,
```

**Software Installation Audit Log**: Records agent decisions, tool calls, escalations, notifications, and human actions.

- `event_id` : Text - Unique audit event identifier.
- `request_id` : Text - Related request identifier.
- `actor` : Text - Agent, requester, approver, or integration name.
- `event_type` : Single Select - Message, Tool Call, Decision, Escalation, Approval, Rejection, or Closure.
- `details` : JSON - Structured event details.
- `created_at` : Date - Event timestamp.

```csv
event_id,request_id,actor,event_type,details,created_at
EVT-2048,IT-1048,Software Installation Request Agent,Escalation,"{""reason"":""Policy requires owner review""}",2026-05-18T09:18:00.000Z
```

**Software Install Request Form**: A structured request screen for employees who need to provide required fields, attachments, or target record details that are awkward to collect in chat.

**Software Review Queue**: A queue for approvers and operations owners to review escalated requests, see the agent's rationale, and record a final decision.

**Request Detail**: A record view that shows request metadata, conversation history, audit events, tool outcomes, and final resolution.

<h2 id="automations">Automations</h2>

**Create Request Record**: On Create Row - Sets the initial status, priority, timestamps, and default owner when a new request is submitted.

**Invoke Software Installation Request Agent**: On Create Row - Sends the request context to the agent for triage and policy evaluation.

**Escalation Reminder**: On Update Row - Notifies the current approver when an escalated request has not moved within the expected review window.

**Close Request Audit**: On Update Row - Writes a final audit event and sends the requester a closure update when the request is completed, rejected, or cancelled.
